Return the code based on the strongest likely buying interest, content appeal, category behavior, and feed relevance.

Use only when the product has no meaningful male/female/home/kids leaning.
Examples: generic phone cables, basic chargers, simple electronics, neutral office items, generic storage boxes, general cleaning refills.

2 = Male-leaning interest feed.

Examples: motorcycle accessories, car performance accessories, car tools, garage equipment, workshop tools, tactical-looking gear, repair tools, heavy DIY tools, fishing gear, camping gear, tech gadgets with masculine/performance styling, men's grooming, men's fashion.

4 = Couple, family, or shared home feed.
Use for products mainly bought for shared home life, family comfort, home organization, bedding, general kitchen/home use, couple gifts, home decor that is not clearly feminine-styled.

5 = Kids, babies, or parents feed.

It means "would this product normally feel like a thoughtful, socially natural gift".

The product should feel personal, enjoyable, beautiful, comforting, fun, meaningful, decorative, surprising, pampering, lifestyle-related, hobby-related, or clearly considerate.

Set is_giftable = 0 if the product is mainly functional, technical, generic, ordinary, compatibility-dependent, maintenance-related, repair-related, replacement-related, or something people usually buy only because they currently need it.

Before deciding, silently judge the product by these questions:

* Would this feel natural as a birthday gift, holiday gift, small surprise, wishlist item, or thoughtful present?
* Would the recipient feel that someone thought about them personally?
* Does the product create delight, comfort, style, emotion, fun, beauty, or a meaningful personal benefit?
* Is it more than just a practical item needed for a technical task?

If the gift value is weak, unclear, purely practical, or only relevant when someone specifically needs that exact item, return 0.

Be strict.
Prefer 0 when uncertain.
Return only 0 or 1.
